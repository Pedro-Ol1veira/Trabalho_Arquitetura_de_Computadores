Fluxo de Cálculos do GPf

Pilha em Dados (Fluxo de Cálculo)


                  // Cálculos para o cenário "Pilha em Dados":
                  // 1. Endereço Base do Segmento de Dados é usado como referência.
                  // 2. A Pilha é alocada a partir de um offset dentro do segmento de Dados.
                  //    Ex: Endereço_Pilha = Endereço_Dados + Offset_Aleatório.
                  // 3. O crescimento da Pilha ocorre em direção aos endereços menores (decrescente).
                  //    Push: Decrementa SP, copia dado.
                  //    Pop: Copia dado, incrementa SP.
                  //
                  // Visualização:
                  // [ Segmento de Código ] [ Segmento de Dados (contendo Pilha) ] [ Segmento Extra ]
                  //                           ^Pilha cresce para cá

Este fluxo descreve como a pilha é mapeada e operada quando localizada dentro do segmento de dados.

Pilha em Código (Fluxo de Cálculo)


                  // Cálculos para o cenário "Pilha em Código":
                  // 1. Endereço Base do Segmento de Código é usado como referência.
                  // 2. A Pilha é alocada a partir de um offset dentro do segmento de Código.
                  //    Ex: Endereço_Pilha = Endereço_Código + Offset_Aleatório.
                  // 3. Crescimento da Pilha decrescente.
                  //
                  // Visualização:
                  // [ Segmento de Código (contendo Pilha) ] [ Segmento de Dados ] [ Segmento Extra ]
                  //          ^Pilha cresce para cá
                  //
                  // Nota: Este cenário é incomum e pode levar a falhas de proteção (GPf)
                  // em sistemas modernos devido a permissões de execução.

Este fluxo mostra o mapeamento e operação da pilha quando ela compartilha o espaço do segmento de código.

Código em Pilha (Fluxo de Cálculo)


                  // Cálculos para o cenário "Código em Pilha":
                  // 1. Endereço Base do Segmento de Pilha é usado como referência.
                  // 2. O Código executável (instruções) é carregado na Pilha.
                  //    Ex: Endereço_Código_Executável = Endereço_Pilha + Offset.
                  // 3. O Program Counter (PC) aponta para o endereço do código na pilha.
                  //
                  // Visualização:
                  // [ Segmento de Código Original ] [ Segmento de Dados ] [ Segmento de Pilha (contendo Código) ]
                  //                                                              ^PC aponta para cá
                  //
                  // Nota: Este cenário é uma **GRANDE VULNERABILIDADE DE SEGURANÇA**
                  // (como buffer overflows que injetam código) e é evitado. Resultaria em GPf.

Este fluxo ilustra um cenário onde o código é executado diretamente da pilha, o que é uma prática extremamente insegura.
